The Project

This project solves a major need in the College of Computer Science and Engineering. For years now, the CCSE department has been expending a tedious amount of time and effort to keep up with all the documentation and status all the projects in the college. This project offers a much-needed solution. Now, instead of storing project information haphazardly and on one machine, projects will be stored on KSU servers and can be accessed from anywhere around the globe.

The Dashboard

This page allows for easy access to creating a new project as well as displaying current project data that can be understood at a quick glance. The pie chart is a depiction of all projects by category. As long as they're not deleted, they will show up as data here. In the line graph the user is presented with the number of projects created each month for the last year.

The Projects

Projects are this capstone's backbone.

Project View (Top)

Project View (Bottom)

The ability for users to add, edit, and delete projects is a pivotal feature. When creating or editing a project, there are 4 main sections. The first is a description. This step is consistent for every project. The same questions are asked no matter what project type. These are questions like the project's name, who the sponsor is, and a description.

Description Questions

This next page is the highlight of the project. This page displays all the questions for the project. Each question is configurable and customizable. You can read more about that in the "Forms" highlight.

Form Questions

Documents are another important part to any project. This allows NDAs and signed contracts to be uploaded and stored for future reference.

Upload Documents

Last, but not least, is the ability to add members to a project. This grants the user access to easily email members of a project individually or to email the group as a whole when viewing the project.

Add Members

The Forms

Customizable forms is what makes our project different than others. Fully modifiable forms were a priority from the very beginning. They were not asked for, but our team decided that for a project of this scale and importance, having the ability to alter text in a contract or update questions asked in a form were of critical importance.

Each question has a 'content' area where the main question goes and a 'sub-content' area where the user may add additional and clarifying information about the question. The drop-down input allows the user to select what question type to use. Some options are text, phone, email, checkbox, etc.
